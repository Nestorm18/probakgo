### Building

```bash
# Server (API + web UI)
go build -o probaky .
./probaky          # reads .env, listens on port 36748

# Client (Proxmox monitoring agent)
go build -o probaky-client ./client/
```

### What's implemented

**Server:**
- API: all endpoints (health, auth, PVE/PBS reports, backup config, admin API keys, download)
- Web UI: dashboard, PVE servers + detail + historical reports, PBS servers, API keys, users, email settings, profile
- Email: daily report scheduler (configurable time via DB, SMTP with STARTTLS)
- DB: SQLite with embedded migrations (no Alembic)
- Auth: bcrypt passwords, session cookies (gorilla/sessions)
- Roles: 3-tier RBAC — `reader` (read-only), `editor` (backup config), `admin` (full access)
- API key types: `pbk-` (client), `app-` (mobile), `adm-` (admin)

**Client:**
- Detects server type (PVE/PBS) from `/etc/issue`
- PVE: queries storages, backup tasks, content; sends to `POST /report/pve`
- PBS: queries datastore usage; sends to `POST /report/pbs`
- Machine ID binding via `/etc/machine-id`
- TLS: configurable verify/skip/CA bundle via env vars
- vzdump hook mode: `--vzdump-hook` (called by Proxmox after each backup)
- File mode: `--file path.json` for testing without a live Proxmox node

## Project Overview

**probaky** monitors Proxmox Virtual Environment (PVE) and Proxmox Backup Server (PBS) nodes and provides a web dashboard with backup status, storage usage, and email reports.

### Component interaction

```
probaky-client  ──POST /report/pve──▶  probaky (server)
probaky-client  ──POST /report/pbs──▶  probaky (server)
                                            │
                                       SQLite DB
                                            │
                               Web UI (browser ← port 36748)
```

### Client installation on a Proxmox node

```bash
# 1. Build and copy binary to the node
go build -o probaky-client ./client/
scp probaky-client root@proxmox-node:/tmp/

# 2. On the node, run the install script
scp scripts/install_client.sh scripts/vzdump_client.sh root@proxmox-node:/tmp/
ssh root@proxmox-node "bash /tmp/install_client.sh --api-url http://your-server:36748 --api-key pbk-..."

# 3. Verify
ssh root@proxmox-node "/opt/probaky/probaky-client"
```

The script:
- Copies binary to `/opt/probaky/`
- Auto-generates Proxmox API token via `pveum` (PVE) or `proxmox-backup-manager` (PBS)
- Writes `/opt/probaky/.env`
- Installs vzdump hook in `/etc/vzdump.conf`
- Configures logrotate

**Updates** (CI/CD): just copy the new binary to `/opt/probaky/probaky-client`. No service restart needed — the client is invoked per backup, not as a daemon.

### Client configuration (`/opt/probaky/.env`)

```env
API_KEY=pbk-...                        # from Probaky web UI
API_URL=http://your-server:36748
PROXMOX_TOKEN=root@pam!probaky-client  # auto-generated by install script
PROXMOX_SECRET=xxxxxxxx-xxxx-...
PROXMOX_VERIFY_TLS=false               # false for self-signed certs (default on Proxmox)
# PROXMOX_CA_BUNDLE=/path/to/ca.pem   # optional: custom CA
```

| `editor` | reader + backup config editing |
| `admin` | Full access including users, API keys, email settings |
